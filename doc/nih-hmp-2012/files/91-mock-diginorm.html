Running digital normalization on the HMP mock data set

This notebook runs 'digital normalization' (see http://ged.msu.edu/papers/2012-diginorm/) on the HMP Illumina mock data set.

Basically, it starts with two gzipped fastq files and runs a four-pass approach:

  1. Normalize to coverage = 20
  2. Trim low-abundance (likely erroneous) k-mers
  3. Trim high-abundance (likely Illumina artifacts)
  4. Normalize to coverage = 5

At the end, the fraction of reads tossed are graphed.

This produces files that can be used as input into the partitioning algorithm (see the next notebook!)

Here we are starting with the two fastq.gz files, raw, untrimmed, etc. etc.
